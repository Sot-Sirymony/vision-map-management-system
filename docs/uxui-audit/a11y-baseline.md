# Runtime Accessibility Baseline — FR-19.4 (M-4)

**Date:** 2026-07-18
**Instrument:** axe-core 4.x (the same engine behind Lighthouse's
accessibility category) via Playwright/Chromium, 1440×900, seeded test
account. Rule set: WCAG 2.0/2.1/2.2 A + AA + best practices. Raw output:
[screens/axe-results.json](screens/axe-results.json). Re-measure at release
close with the same instrument (FR-19.6).

## Result after FR-26 (2026-07-18)

**Zero axe violations (WCAG A/AA + best practices) on all six top pages, in
both themes** — and zero on sampled accent variants (orange light/dark,
green light, purple dark). Raw after-data: `axe-after-fr26.json` in the
audit scratchpad. What closed the gap: landmarks (`<nav>` sidebar, explicit
`<header>`, existing `<main>`), accessible names on every MUI Select
(SelectDisplayProps) and Autocomplete filter, theme-aware chip text
(`color-mix` toward the foreground token), darker faint-text tokens, a
theme-aware moonshot foreground token, ARIA-valid tree structure with
labeled quick-add inputs, per-card "Move to status" names plus a live
region announcing board moves, breadcrumbs demoted from landmarks, 24px
legend targets, and a global `:focus-visible` ring.

## Visual verification notes (from the screenshot sweep)

- **Dark mode holds up** overall — the V-08 concern (hardcoded light tints in
  dashboard cards) renders acceptably, though the tints are still token
  drift and should move to the theme.
- **Confirmed in dark mode:** the "…of your goals is a moonshot" line
  (hardcoded `#6b21a8` purple) is barely legible; chip labels
  (CRITICAL/BLOCKED etc.) are among the 102 contrast nodes.
- **FR-18 combination sweep: no layout breakage.** The worst combination
  (dark + compact + large text + orange accent) renders correctly on all
  top-6 pages — accent flows through buttons, chips, progress bars, and
  nav; the compact density and large text compose without overflow.
  Baseline for the "zero open combination-sweep bugs" criterion: **0 layout
  bugs, contrast issues as counted above.**
- Overdue tasks in the Dashboard "Priority tasks" table carry **no** overdue
  marking at all (not even the color-only row tint the list pages have) —
  extends hci-audit H-04.
- Raw ISO dates confirmed on screen (`2026-07-15` in Priority tasks).
